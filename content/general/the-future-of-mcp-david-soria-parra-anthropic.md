## 1. 핵심 주제
- **미래의 MCP (멀티-커넥티비티 프로토콜):** MCP의 발전과 이를 통한 에이전트 구축의 미래 전망

## 2. 주요 개념 정리
- **MCP 정의:** 클라이언트와 서버 간 상호작용을 지원하는 프로토콜로, 애플리케이션과 툴을 함께 배포 가능
- **작동 원리:** 클라우드나 다양한 플랫폼에 MCP 서버를 배치하여 플랫폼 독립적으로 애플리케이션과 툴 제공
- **중요한 주장:** MCP는 다양한 연결성을 제공하여 지식 작업을 수행하는 에이전트 개발에 필요
- **근거:** 110만 다운로드 달성, 여러 SDK와 프레임워크에서 활용

## 3. 세부 설명
- **MCP의 진화:** 초기에는 로컬에서만 사용되던 MCP가 이제는 원격 기능과 중앙화된 인증을 추가
- **에이전트의 필요성:** 2026년에는 복잡한 작업을 수행할 수 있는 에이전트가 필요하며, MCP는 이러한 에이전트의 중요한 연결 수단
- **구성 요소:** 에이전트 구축 시 고려해야 할 세 가지 주요 요소는 스킬, MCP, CLI/컴퓨터 사용
- **연결성:** 에이전트는 다수의 SaaS 애플리케이션과 공유 드라이브에 연결 가능해야 함

## 4. 예시 / 적용
- **도구 검색 및 프로그래밍 도구 호출:** MCP는 필요할 때 도구를 로드하는 방식으로 컨텍스트 크기를 줄임
- **구조적 출력 사용:** 모델이 출력의 유형 정보를 활용하여 작업을 효율적으로 조합

## 5. 시사점
- **에이전트의 발전:** MCP는 다수의 플랫폼과 시스템을 연결하여 다양한 분야의 지식 작업을 자동화하는 데 필수적
- **미래 방향:** 에이전트 구축 시 MCP를 포함한 다양한 연결 수단을 통합적으로 활용해야 함

## Timeline

**Core Process:**

1. **MCP 개요 및 중요성 이해:**
   MCP는 클라이언트 및 서버 측에서 인터페이스를 제공하는 에이전트로 설명됩니다. 이는 클라우드, ChatGPT, VS Code Cursor 등 다양한 플랫폼에 배치할 수 있으며, 사용자와 모델이 상호작용할 수 있는 도구를 함께 제공하여 독특한 경험을 제공합니다.

2. **MCP 개발의 역사와 발전:**
   초기 스펙 문서부터 시작해 18개월 동안 MCP는 로컬 도구에서 원격 기능 및 다양한 기능이 추가되면서 급속히 발전했습니다. 현재는 월 1억 1천만 다운로드를 기록하며 다양한 서버와 통합하여 사용되고 있습니다.

3. **에이전트의 미래와 필요 요소:**
   2026년에는 다양한 SaaS 애플리케이션과 연결할 수 있는 에이전트가 필요하며, 연결성은 에이전트 개발의 핵심 요소로 강조됩니다. 다양한 도구와 기술을 활용하여 에이전트를 구축하는 방법이 논의됩니다.

4. **프로그레시브 디스커버리와 프로그램적 도구 호출:**
   MCP 프로토콜의 활용을 극대화하기 위해, 도구의 로딩을 지연시키고 필요할 때 로딩하는 패턴을 도입합니다. 이는 모델이 도구를 효율적으로 사용할 수 있도록 돕습니다.
