## Description

## 1. 핵심 주제

클로드 코드의 사용 시 발생하는 성능 저하 문제와 이를 해결하기 위한 설정 및 도구를 소개합니다. 특히 비용을 절감하고 성능을 최적화하는 방법에 대해 알아봅니다.

## 2. 주요 개념 정리

### 컨텍스트 로트 (Context Rot)
- **정의**: 장시간 사용 시 클로드 코드의 성능이 서서히 저하되는 현상.
- **원인**:
  1. **어텐션 희석**: 문맥이 길어질수록 중요 정보를 놓침.
  2. **명령 충돌**: 모순된 지시문으로 인해 혼란 발생.
  3. **토큰 예산 압박**: 불필요한 정보가 메모리를 차지.
  4. **관련성 미스 매치**: 불필요한 정보가 작업에 포함.

### 비용 절감 설정 (`settings.json`)
- **모델 변경**: Opus 대신 Sone 사용으로 약 60% 절감.
- **Max Thinking Tokens 감소**: 31,999에서 10,000으로 줄이면 70% 절감.
- **Sub Agent 모델**: 기본 모델을 Haiku로 변경해 추가 절감.

### 메모리 학습 시스템
- **정의**: 작업 패턴을 자동으로 학습하여 재사용.
- **작동 원리**: 반복되는 작업 패턴을 Instinct로 저장하고, 신뢰도 점수를 부여.
- **응용**: Instinct를 모아 자동 스킬 모듈로 활용 가능.

## 3. 세부 설명

### 컨텍스트 로트의 작동 원리
- 클로드 코드가 장시간 사용되면 정보의 중복과 혼란으로 인해 성능이 떨어진다.
- 비유적으로, 20만 토큰의 연료탱크가 있지만 불필요한 짐으로 인해 효율이 떨어진다.

### 비용 절감 설정의 적용
- 홈 디렉토리의 `settings.json` 파일에 세 줄 추가.
- 모델 변경, 토큰 감소, 서브 에이전트 최적화를 통해 비용 절감 가능.

### 메모리 학습 시스템의 중요성
- 지속 학습을 통해 코드 작성 습관을 파악하고 자동으로 스킬 모듈화.
- 작업 효율성을 높이고, 팀 내 지식을 공유할 수 있음.

## 4. 예시 / 적용

### 실제 적용 사례
- **모델 변경 예시**: 복잡한 디버깅이 필요할 때만 Opus로 전환.
- **메모리 학습 시스템**: 회사의 코딩 컨벤션을 자동 학습하여 신입에게 전수.

## 5. 시사점

- 클로드 코드의 성능 저하는 구조적 문제로, 이를 해결하기 위한 최적화 도구가 중요하다.
- 설정 변경만으로도 큰 비용 절감 효과를 얻을 수 있으며, 지속적인 학습 시스템은 장기적으로 효율성을 높인다.

## 핵심 정리

- **컨텍스트 로트**: 클로드 코드 성능 저하의 구조적 문제.
- **비용 절감 설정**: `settings.json`에 모델, 토큰, 서브 에이전트 최적화.
- **메모리 학습 시스템**: 작업 패턴 자동 학습 및 재사용.
- **슬래시 명령어**: `/clear`와 `/compact`의 차이 이해.
- **MCP 서버 관리**: 활성 도구 수 제한으로 효율성 유지.
- **에이전트 쉴드**: 보안 스캐너로 설정 검증.
- **시작 방법**: 설정 변경 후 필요에 따라 도구 및 스킬 추가.
