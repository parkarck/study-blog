# 하네스 엔지니어링 개념 + 적용사례를 소개합니다.



## 1. 핵심 주제

하네스 엔지니어링은 복잡한 코딩 작업을 효과적으로 관리하고 해결하기 위한 엔지니어링 접근 방식입니다. 이 방식은 기존의 코딩 에이전트의 한계를 극복하고, 복잡한 문제를 체계적으로 분해하여 해결하는데 중점을 둡니다.



## 2. 주요 개념 정리

- **하네스 엔지니어링**: 복잡하고 큰 프로젝트를 잘게 쪼개어 접근하는 방법론. 구현과 검증을 담당하는 에이전트를 나누어 협업을 촉진합니다.

- **코딩 에이전트의 한계**: 컨텍스트 크기에 의해 작업이 제한되는 문제점.

- **협업 구조**: 여러 에이전트를 통한 역할 분담으로 작업의 품질과 효율성 향상.

- **핵심 기능**:

  - **스킬**: 작업에 필요한 프롬프트, 라이브러리 사용법 등을 저장.

  - **서브에이전트**: 특정 작업을 하위 에이전트에 위임.

  - **포크**: 대화 흐름을 복제하여 같은 컨텍스트로 작업을 지속.

### 코딩 에이전트의 한계

### 핵심 기능

- **서브에이전트**: 특정 작업을 하위 에이전트에 위임하여 작업을 분산.
