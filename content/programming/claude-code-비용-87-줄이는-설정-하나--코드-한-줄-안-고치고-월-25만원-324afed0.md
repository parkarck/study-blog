# Claude Code 비용 87% 줄이는 설정 하나 | 코드 한 줄 안 고치고 월 25만원 절약



## 1. 핵심 주제

- Claude Code의 모델 오버라이드 설정을 통해 비용을 효과적으로 절감하는 방법



## 2. 주요 개념 정리



### 모델 오버라이드

- **정의**: 다양한 작업에 맞춰 여러 모델을 선택적으로 사용하는 방법

- **작동 원리**: 파일의 유형 및 작업의 복잡도에 따라 적합한 모델을 사용하여 비용을 줄임

- **중요한 주장**: 동일한 결과를 얻으면서도 비용을 최대 87%까지 절감 가능



### 비용 절감 전략

- **정의**: 특정 작업에 가장 적합한 모델을 선택하여 토큰 사용량을 최적화

- **작동 원리**: 하이쿠, 순넷, 오퍼스 등의 모델을 각 작업에 맞게 분배

- **중요한 주장**: 설정 하나로 월 25만원 절약 가능



## 3. 세부 설명



### 모델 오버라이드 적용 방법

1. **모델 분류**:

   - **하이쿠(Haiku)**: 간단한 파일 작업 및 테스트에 사용

   - **순넷(Sonnet)**: 코드 점검 및 수정에 적합

   - **오퍼스(Opus)**: 아키텍처 설계에 활용

   

2. **설정 과정**:

   - JSON 파일에 모델 설정 적용

   - 프로젝트별 혹은 글로벌 설정 가능



3. **비용 비교**:

   - 모든 작업에 오퍼스를 사용할 경우 비용이 5배 증가

   - 하이쿠와 순넷을 적절히 활용할 경우 비용이 평균 1.6배로 절감



## 4. 예시 / 적용



### 예시 설정 적용

- **하이쿠**: 파일 검색 시 사용

- **순넷**: 코드 리뷰 및 종합 분석

- **오퍼스**: 복잡한 아키텍처 설계 적용



### 실제 적용 사례

- 다양한 프로젝트에서 병렬로 모델을 적용하여 토큰 사용량과 비용 절감 효과 확인



## 5. 시사점

- 모델 오버라이드는 코드 변경 없이 비용 효율성을 극대화할 수 있는 강력한 도구

- 프로젝트의 요구 사항에 따라 적절한 모델을 선택함으로써 비용 절감과 효율성을 동시에 달성 가능

## 기타 기능 소개



### 에이전트 모델 파라미터 활용

- 프롬프트에서 원하는 모델을 직접 지정하는 방법

- 작업의 성격에 따라 Haiku/Sonnet/Opus를 분리하여 사용



### /loop 활용법

- 반복 모니터링 및 점검 작업을 자동화하는 방법

- 예: 매일 특정 시간에 데이터 확인 및 업데이트 자동화



### /cost 확인법

- 실제 비용 절감 규모를 확인하는 방법

- `/slash` 명령어로 비용 체크 가능
