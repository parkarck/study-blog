## Claude Code 대격변, 스킬이 앱이 됐습니다!



### 핵심 주제

Skills 2.0이 단순한 도구에서 소프트웨어/애플리케이션으로 진화했습니다. Anthropic이 이를 선언했습니다.



### 5가지 핵심 기능



1. **격리 컨텍스트 실행 (Context Fork)**

   - 스킬이 메인 컨텍스트와 분리되어 독립적으로 실행

   - 컨텍스트 윈도우 효율성 향상



2. **스킬별 훅 라이프사이클**

   - 각 스킬마다 독립적인 훅 설정 가능

   - 글로벌 훅 대신 스킬별 개입 조절



3. **자가 검증 (Evals)**

   - AI가 스킬 품질을 자동으로 검증

   - 정량적 보고 제공

   - 개발자: 내 스킬이 제대로 동작하는지 확인

   - 사용자: 남이 만든 스킬 품질 평가



4. **스킬 생성 자동화**

   - Skill Creator 도구로 자연어로 스킬 생성

   - 자동으로 테스트 및 평가까지 완료

   - 생성 시간: 30분-2시간 → 10-30분으로 단축



5. **스킬 수명 관리 (Classification)**

   - 워크플로우 장착 (예: PDCA)

   - AI 모델 발전과 무관하게 프로세스 유지

   - 스킬 자체가 엑셀, 파워포인트처럼 독립적 컴포넌트화



### 사용자 경험 변화



- **Hot Reload**: 스킬 수정 시 세션 재시작 불필요

- **실시간 반영**: 수정 사항 즉시 적용

- **품질 확신**: 자가 검증으로 스킬 동작 확인 가능



### AI Native 방법론



**3대 원칙:**

1. 사람이 해야 할 일과 AI가 해야 할 일 명확히 나누기

2. 사람은 AI 결과물에 책임지기

3. 자동화 (Automation First), 문서화 (Documentation is Code), 검증 (Eval)



**핵심:**

- 모델 성능이 아니라 **워크플로우가 경쟁력**

- 컨텍스트 엔지니어링이 핵심

- 장비보다 빌드/전략이 중요함 (스타크래프트 비유)



### Bkit 통합



- 28개 프론트 메타 모든 스킬에 적용

- 훅 5계층 시스템으로 최적화

- PDCA, BT Rule 등 다양한 워크플로우 지원



### 결론



Skills 2.0의 본질은 **소프트웨어 방법론**입니다. AI 모델이 발전할수록 우리는 **프로세스의 차별화**에 집중해야 합니다.
