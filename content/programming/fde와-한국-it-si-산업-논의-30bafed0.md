# FDE와 한국 IT SI 산업 논의

**Category:** programming

---

[FDE와 한국 IT SI 산업 논의 - 2026-02-18]



## 1. 핵심 주제



- **FDE(Fully Deployed Engineer)의 개념**: FDE의 역할과 책임, 그리고 한국 IT SI 산업과의 비교

- **한국 IT SI 산업의 특성**: 하청 구조와 요구사항 정의 문제, 솔루션 커스터마이징의 한계



## 2. 주요 개념 정리



### FDE (Fully Deployed Engineer)

- **정의**: FDE는 특정 프로젝트 팀에 완전히 배치되어 팀의 일원으로서 무한 책임을 지고 작업하는 엔지니어.

- **작동 원리**: 팀 단위로 움직이며 직접적이고 통합된 접근 방식을 취함. 회사 내에서 독립적이면서도 책임을 지는 구조.

- **중요한 주장**: FDE는 단순한 월급받는 엔지니어와 다르며, 프로젝트의 성공을 위해 더 깊은 책임을 지님.



### 한국 IT SI 산업

- **정의**: 시스템 통합(System Integration) 사업을 중심으로 IT 솔루션과 서비스를 제공하는 산업.

- **특성**: 하청업체에 책임 전가, 고객의 요구사항 정의 부족, 외산 솔루션 커스터마이징에 대한 높은 의존도.

- **중요한 주장**: 비용 절감과 맞춤형 솔루션 요구로 인해 외산 솔루션의 실패 가능성이 높음.



## 3. 세부 설명



### FDE의 역할과 책임

- FDE는 팀에 완전히 통합되어 프로젝트 관리 및 데이터 분석을 수행.

- 엔지니어링 관점보다 비즈니스 요구에 맞춘 데이터 분석과 모델 개발에 초점을 맞춤.

- 예: 팔란티어의 FDE는 풀스택 개념으로 접근하여 팀의 모든 영역을 책임짐.



### 한국 IT SI 산업의 문제점

- **하청 구조**: 하청업체에게 업무와 책임을 전가하는 구조적 문제.

- **요구사항 정의 부족**: 고객이 명확한 요구사항을 제공하지 못하는 경우가 많아 분석/설계자가 요구사항 정의까지 수행해야 하는 상황.

- **솔루션 커스터마이징**: SAP나 오라클 같은 독점 솔루션을 제외하고는 커스터마이징 요구가 큼. 



## 4. 예시 / 적용



- **팔란티어의 FDE**: 팀 단위로 운영되며, 회사 자체가 팀을 구성하여 무한책임 구조를 형성.

- **현대캐피탈 프로젝트 사례**: 프랑스 BA가 고객에게 시스템에 대한 이해 부족을 지적한 사례.

- **와이즈넛과 솔트룩스**: 와이즈넛은 공공기관 위주로 납품하며 전략적 수익을 창출. 반면 솔트룩스는 엔지니어링 접근으로 인해 재정적인 어려움에 직면.



## 5. 시사점



- **FDE의 중요성**: 프로젝트 성공을 위해 FDE와 같은 책임감 있는 엔지니어의 필요성 증가.

- **한국 IT SI 산업의 개선 필요**: 하청 구조 개선과 명확한 요구사항 정의 절차 확립 필요.

- **솔루션 커스터마이징의 한계**: 맞춤형 요구가 많은 한국 시장에서 외산 솔루션의 실패 가능성 인식 필요.

- **비용 절감과 책임 있는 솔루션**: 저비용 구조와 책임 있는 솔루션 제공 전략의 필요성.



## 핵심 정리



- FDE는 프로젝트의 성공을 위한 팀 단위의 책임 구조를 지닌 엔지니어.

- 한국 IT SI 산업은 하청과 잘못된 요구사항 정의로 인한 문제를 안고 있음.

- 팔란티어의 FDE 모델은 책임 있는 프로젝트 수행에 유리.

- 외산 솔루션은 한국에서 커스터마이징 요구로 인해 실패할 가능성이 높음.

- 와이즈넛과 솔트룩스의 사례는 전략적 접근의 중요성을 보여줌.

- 책임 있는 솔루션과 명확한 요구사항 정의가 산업 발전에 필수적.
