# Advent of Claude - 31일간의 Claude Code 팁

## 1. 핵심 주제
- **주제**: Claude Code를 효율적으로 사용하는 방법
- **목적**: Claude Code 사용법을 익혀서 생산성을 높이는 기법
- **타겟**: 프로그래머, 개발자, 그리고 Claude Code 사용자
- **관련 기술**: 프로그래밍, AI 코딩, 텍스트 에디터, 세션 관리, 플러그인, LSP 통합
- **적용 분야**: 개발자 워크플로우 자동화, AI 네이티브 컴퍼니 GPTers

## 2. 이론 배경
- **Claude Code 소개**: Claude Code 사용법의 체계적 이해와 효율적 사용 방법
- **단축키 기술**: 쉘빌 Bash 단축키 사용법
- **이스케이프 키 기술**: 이스케이프 키로 효율적으로 이동하는 방법
- **세션 관리 기술**: 현재 세션 유지와 전환 자동화
- **개발 환경 설정**: 로컬에서 개발 환경 구축
- **자동화 기술**: 스크립트와 도구로 개발 프로세스 자동화
- **플러그인 기술**: 플러그인으로 기능 확장
- **LSP 통합 기술**: IDE와 유사한 코딩 경험

## 3. 세부 설정

### 3.1 Claude Code 단축키
- **기능**: 단축키로 빠르게 명령어 실행
- **장점**: 키보드 사용 효율성 향상
- **자동화**: 반복 작업 자동화

### 3.2 이스케이프 키
- **! Prefix**: 쉘빌 Bash 단축키 모드
- **Double Esc**: 이모지 키 조합으로 탈출할 수 없게
- **Ctrl + R**: 폰트 단축키 재로드
- **Prompt Stashing**: Ctrl+S로 저장
- **Prompt Suggestions**: 탭 키 자동완성

### 3.3 세션 관리
- **Continue Where You Left Off**: 마지막 작업 위치로 이동
- **Named Sessions**: 세션 이름으로 구분 관리
- **Claude Code Remote**: 원격 작업 시 세션 유지

### 3.4 개발 환경
- **Vim Mode**: Vim 단축키로 코딩 경험
- **/statusline**: 개발 진행 상황 한눈에 확인
- **/context**: 현재 작업 내역 요약
- **/stats**: 프로젝트 통계 대시보드
- **/usage**: 프로젝트 사용량 한눈에 확인

### 3.5 자동화
- **Subagents**: 각 에이전트가 특정 작업을 처리
- **Agent Skills**: 단축키, 이모지, 플러그인, 도구 사용
- **Plugins**: 단축키, 익스텐션, 오토매팅, 프리셋, 파일 제어
- **Language Server Protocol (LSP) Integration**: IDE와 유사한 코딩 경험

## 4. 이론/실습
- **코드 테스트 설정**: /init 명령어를 입력해 CLAUDE.md 파일 생성
- **단축키 사용법**: ! Prefix를 입력해 Bash 단축키 모드 실행
- **개발 도메인 전환**: /stats 명령어로 프로젝트 도메인 전환
- **성능 대시보드**: /stats와 /usage로 프로젝트 통계 확인

## 5. 이론/실습
Claude Code를 효율적으로 사용하여 개발 생산성을 극대화하는 방법론과 실습입니다. 이 가이드라인은 단순히 명령어 정리가 아니라, 개발자가 AI와 협력하여 워크플로우를 완전히 자동화하는 방법론입니다.
