# [메타 엔지니어가 알려주는 에이전틱 엔지니어링 핵심 개념5가지 총정리]



## 1. 핵심 주제

에이전틱 엔지니어링(Agentic Engineering)은 인공지능 에이전트를 효과적으로 설계하고 운영하는 방법론으로, 에이전트의 자율성과 최적의 성능을 발휘할 수 있도록 하는 데 중점을 둡니다.



## 2. 주요 개념 정리



1. **에이전트(Agent)**

   - **정의**: 목표를 해석하고 계획을 세우며 도구를 사용하여 자율적으로 실행하는 주체.

   - **작동 원리**: 목표 해석 → 계획 수립 → 실행 → 평가 루프.

   - **중요한 주장**: 단순한 챗봇과 달리 에이전트는 자율성과 도구 활용 능력이 뛰어남.



2. **컨텍스트 엔지니어링(Context Engineering)**

   - **정의**: 에이전트의 두뇌를 설계하는 것으로, 어떤 정보를 언제 어떻게 제공할지에 집중.

   - **중요한 주장**: 정보의 양과 질에 따라 결과물이 달라짐.



3. **멀티 에이전트 오케스트레이션(Multi-Agent Orchestration)**

   - **정의**: 여러 에이전트를 조직화하여 복잡한 작업을 수행.

   - **작동 원리**: 메인 에이전트가 오케스트레이터 역할을 하여 서브 에이전트들에게 작업 분배.

   - **중요한 주장**: 복잡한 작업을 효과적으로 처리하기 위해 필수적.



4. **훅과 스케일(Hook and Scale)**

   - **정의**: 에이전트가 외부 세계와 상호작용하는 수단.

   - **중요한 주장**: 다양한 도구와 데이터 소스를 활용하여 에이전트의 유연성과 확장성을 높임.



5. **플래닝(Planning)**

   - **정의**: 에이전트가 목표를 달성하기 위한 구체적인 계획을 수립하는 과정.

   - **중요한 주장**: 철저한 계획은 에이전트의 성공적인 목표 달성을 가능하게 함.



## 3. 세부 설명



### 에이전트와 챗봇의 차이

- **에이전트**: 목표 지향적, 도구 활용 가능, 자율적 오류 수정.

- **챗봇**: 질문-답변 기반, 도구 활용 불가, 사용자가 직접 지시 필요.



### 컨텍스트의 중요성

- **비용과 지연**: 컨텍스트가 커질수록 비용과 응답 시간 증가.

- **신호 약화**: 과도한 정보는 핵심을 놓치게 만듦.

- **캐시 효율성**: 반복적 정보 제공 시 비용 절감 가능.



### 멀티 에이전트 시스템

- **구조**: 메인 에이전트(오케스트레이터) + 서브 에이전트

- **효과**: 병렬 작업과 결과 통합으로 효율성 극대화.



## 4. 예시 / 적용



- **블로그 플랫폼 개발**: 에이전트는 단순히 한 번에 결과를 내놓는 것이 아니라 무엇을 해야 하는지 분석, 순서 정리, 도구 사용 등 단계별로 작업 수행.

- **영화 제작 비유**: 감독(메인 에이전트)이 각 역할(서브 에이전트)을 배치하고 감독하여 최종 작품 완성.



## 5. 시사점



- **에이전틱 엔지니어링**은 AI의 자율성과 효율성을 극대화하여 복잡한 문제 해결에 핵심적인 역할을 함.

- **컨텍스트 설계**는 AI의 성능을 좌우하므로 체계적인 정보 제공이 필수적.

- **멀티 에이전트 오케스트레이션**은 복잡한 프로젝트의 성공적인 운영을 위한 필수 전략.



## 핵심 정리



- **에이전트**는 자율적이고 도구 활용 가능하며, 목표 지향적으로 작동한다.

- **컨텍스트 엔지니어링**은 AI의 두뇌 설계로, 정보의 양과 질이 결과물에 큰 영향을 미친다.

- **멀티 에이전트 오케스트레이션**은 복잡한 작업을 효율적으로 처리하기 위한 구조적 접근법이다.

- **훅과 스케일**은 외부와의 상호작용을 통해 에이전트의 기능을 확장시킨다.

- **플래닝**은 목표 달성을 위한 철저한 계획 수립 과정이다.
