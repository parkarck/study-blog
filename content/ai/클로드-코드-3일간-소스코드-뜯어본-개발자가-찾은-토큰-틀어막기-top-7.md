# 클로드 코드 3일간 소스코드 뜯어본 개발자가 찾은 토큰 틀어막기 TOP 7

## 1. 핵심 주제
클로드 코드를 효과적으로 사용하여 토큰 소비를 줄이는 방법을 찾는 것이 이 강의의 핵심 주제입니다. 이 주제는 클라우드 기반의 코드 실행 환경에서 발생하는 불필요한 토큰 낭비를 줄이고, 효율적인 작업 방식을 도입하여 비용을 절감하는 방법을 설명합니다.

## 2. 주요 개념 정리
- **토큰 관리**: 클라우드 환경에서 사용되는 데이터 단위로, 비용 절감의 핵심.
- **MCP 서버**: 여러 도구를 연결할 수 있는 서버로, 필요하지 않은 경우 꺼야 함.
- **캐싱**: 이전 대화 내용을 저장하여 반복 전송을 줄이는 기술.
- **턴(회차) 재전송**: 대화 내역 전체를 계속 재전송하여 비용 발생.
- **클리어(초기화)**: 불필요한 대화 기록을 제거하여 토큰 낭비 방지.

## 3. 세부 설명
### 3.1 토큰 관리
- **정의**: 클로드에서의 데이터 이동 단위로, 사용량에 따라 비용이 발생.
- **작동 원리**: 대화가 길어지면 이전 대화를 포함하여 모든 내용을 다시 전송, 불필요한 토큰 사용 증가.
- **중요한 주장**: 효율적인 토큰 사용은 비용 절감의 핵심.

### 3.2 MCP 서버 사용
- **정의**: 여러 도구를 연결하여 사용할 수 있는 서버.
- **작동 원리**: 연결된 도구가 많을수록 초기 대화 시 많은 토큰 사용.
- **중요 주장**: 사용하지 않는 서버는 꺼야 함.

### 3.3 캐싱
- **정의**: 이전 대화를 저장하여 반복 전송을 피하는 기술.
- **작동 원리**: 대화를 이어가는 동안 저장, 대화가 끊기면 캐시가 소멸.
- **중요 주장**: 대화를 끊지 않고 몰아서 작업하는 것이 효율적.

### 3.4 턴 재전송
- **정의**: 대화 내역 전체를 매번 처음부터 다시 보내는 방식.
- **작동 원리**: 대화 내용이 매 턴마다 다시 전송됨.
- **중요 주장**: 반복 전송을 줄이는 것이 비용 절감에 중요.

### 3.5 클리어(초기화)
- **정의**: 불필요한 대화 기록을 제거하는 과정.
- **작동 원리**: 대화 내용만 지우고 캐시는 유지.
- **중요 주장**: 주제가 바뀔 때마다 클리어를 해야 함.

## 4. 예시 / 적용
- **MCP 서버 끄기**: 사용하지 않는 도구를 비활성화하여 초기 토큰 사용을 줄임.
- **캐시 활용**: 대화 중간에 끊지 않고 이어서 작업하여 효율적으로 캐시 사용.
- **클리어 사용**: 주제가 바뀔 때마다 초기화하여 불필요한 토큰 사용 방지.

## 5. 시사점
- 효율적인 토큰 관리가 클라우드 환경에서의 비용 절감에 직접적인 영향을 미침.
- 대화 방식 및 작업 습관을 개선하여 더 나은 성과를 낼 수 있음.
- 클라우드 코드 사용 시, 주기적인 검토와 최적화가 필요함.

## 핵심 정리
- 불필요한 MCP 서버 사용을 피하고, 도구 목록을 최적화해야 한다.
- 대화 중간에 끊지 않고 몰아서 작업하면 캐시를 효율적으로 사용할 수 있다.
- 주제가 바뀌면 클리어하여 불필요한 대화 기록을 제거해야 한다.
- 턴을 나눠서 작업하기보다는 한 번에 몰아서 작업하는 것이 비용 절감에 효과적이다.
- 자동 압축 기능은 토큰 사용을 증가시킬 수 있으므로 주의가 필요하다.
- 캐시 및 대화 기록의 작동 원리를 이해하여 적절하게 활용하는 것이 중요하다.
- 최적화된 토큰 사용은 클라우드 서비스의 비용 절감에 큰 기여를 한다.
