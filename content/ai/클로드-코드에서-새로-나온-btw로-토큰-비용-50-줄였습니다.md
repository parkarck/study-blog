## 2. 주요 개념 정리

### 바이더웨이 (BTW)
- **정의**: 클로드 코드에서 사이드 질문을 할 수 있도록 도와주는 명령어.
- **작동 원리**: 메인 작업을 중단하지 않고, 추가적인 세션을 열지 않으며, 기존 세션의 캐시 데이터를 활용하여 효율적인 질문을 가능하게 함.
- **중요한 주장**: 바이더웨이는 토큰 비용을 절반 이상 줄일 수 있으며, 작업의 연속성을 보장합니다.

## 3. 세부 설명

### 바이더웨이의 작동 방식
1. **세션 유지**: 기존 세션의 데이터를 활용하여 추가적인 세션을 열 필요가 없습니다.
2. **캐시 활용**: 이미 로드된 프로젝트 파일과 클로드.md를 그대로 사용하여 새로운 토큰 소비를 방지합니다.
3. **프롬프트 캐싱**: 엔트루픽에서 제공하는 프롬프트 캐싱 기술을 사용하여 입력 비용을 크게 절감합니다.

### 바이더웨이의 주요 기능
